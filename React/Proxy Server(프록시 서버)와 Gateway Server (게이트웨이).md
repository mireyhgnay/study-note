# <div align="center">🖥️ Proxy Server(프록시 서버)와 Gateway Server (게이트웨이)</div>

<br>

### Proxy Server(프록시 서버)

클라이언트와 서버 사이에 중개자 역할을 하는 서버이다.

클라이언트가 자신을 통해서 **다른 네트워크 서비스에 간접적으로 접속**할 수 있게 해 주는 응용 프로그램이다.

**같은 프로토콜을 사용하는 네트워크 간의 통신**이다.

프록시를 사용하면 보안 측면에 이점이나, 캐시 기능으로 성능을 향상시킬 수 있다.

### Gateway Server (게이트웨이)

웹에서 더 복잡한 리소스(동적페이지, 데이터베이스를 통한 콘텐츠 등)가 요구되기 시작하면서 모든 리소스를 한 개의 애플리케이션으로만 처리하기 어려워졌다.

이를 해결하기 위해 게이트웨이를 만든 것이다.

프록시와 같이 클라이언트와 서버 사이의 중개자 역할을 한다.

게이트웨이는 서로 다른 네트워크 상의 통신 프로토콜(protocol,통신규약)을 적절히 **변환**해주는 역할을 한다.

프록시와 가장 큰 차이점은 프로토콜의 변환 여부이다.

- ex) 프록시는 프로토콜이 같은 HTTP 를 사용하는 반면, 게이트웨이는 HTTP, POP 프로토콜을 변환한다.
